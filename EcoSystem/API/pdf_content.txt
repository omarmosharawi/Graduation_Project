THE RE - USER STORIES
============================================================
The RE - User Stories 
Author: Omar Mohamed, Product Manager 
Date: October 16, 2025 
This document outlines the user stories for The RE platform, derived from the features defined 
in the Unified Project Plan V2.0. These stories are centered around our key user persona: the 
environmentally-conscious individual looking for a simple and rewarding way to recycle. 
Epic: User Account & Onboarding 
As a new user, I want a simple and secure way to create and manage my account so that I can 
start recycling and earning rewards quickly. 
● US-001: User Registration 
○ Story: As a new user, I want to sign up for an account using my email address or phone 
number so that I can join The RE community. 
○ Acceptance Criteria: 
■ User can choose to register with an email and password or a phone number. 
■ Password fields must be secure (masked). 
■ Input validation is in place for email format and phone number. 
■ A confirmation/verification step is required to activate the account. 
● US-002: Profile Management 
○ Story: As a registered user, I want to view and edit my profile information so that my 
personal details are always up to date. 
○ Acceptance Criteria: 
■ User can access a dedicated profile screen. 
■ User can update their name and password. 
■ User can view their unique QR code. 
● US-003: Unique QR Code 
○ Story: As a logged-in user, I want to easily access my unique QR code so that I can 
identify myself at a recycling kiosk. 
○ Acceptance Criteria: 
■ The QR code is prominently displayed within my profile or a dedicated section. 
■ The QR code is large and clear enough for a kiosk scanner to read easily. 
Epic: Core Recycling Loop 
As a user, I want a seamless experience from finding a kiosk to depositing my recyclables and 
seeing my points balance update in real-time. 
● US-004: Kiosk Locator 
○ Story: As a user, I want to view an interactive map showing all nearby partner kiosks so 
that I can easily find a place to recycle. 
○ Acceptance Criteria: 
■ The map displays my current location. 
■ Kiosk locations are marked with clear pins. 
■ Tapping a pin shows basic information about the kiosk (e.g., operational hours). 
● US-005: View Points & History 
○ Story: As a user, I want to see my current points balance and a history of my 
transactions so that I can track my recycling efforts and rewards. 
○ Acceptance Criteria: 
■ The main dashboard clearly displays the current redeemable points balance. 
■ A separate screen shows a chronological list of all transactions (points earned and 
redeemed). 
■ Each transaction entry includes the date, location/vendor, and point amount. 
Epic: Rewards & Redemption 
As a user, I want to exchange my recycling points for valuable discounts at partner venues so 
that I feel my efforts are being rewarded. 
● US-006: Rewards Catalog 
○ Story: As a user, I want to browse a catalog of available rewards from partner 
businesses so that I can see what I can redeem with my points. 
○ Acceptance Criteria: 
■ The catalog displays rewards with images, partner names, and the points required. 
■ Users can filter or search for rewards by category or partner name. 
■ Rewards I cannot afford are clearly indicated (e.g., grayed out). 
● US-007: Redeem Reward 
○ Story: As a user, I want to redeem my points for a specific reward so that I can get a 
discount code to use at a partner venue. 
○ Acceptance Criteria: 
■ The redemption process is a simple, two-tap action ("Redeem" -> "Confirm"). 
■ Upon confirmation, my points balance is deducted accordingly. 
■ I receive a unique voucher or code to present to the partner business. 
Epic: Gamification & Engagement (V1) 
As a competitive user, I want to track my overall progress and compete with others so that I am 
motivated to recycle more consistently. 
● US-008: Dual-Point System & Ranks 
○ Story: As a user, I want to have both a "Current Points" balance for rewards and a 
"Total Points" lifetime score so that my overall contribution is recognized. 
○ Acceptance Criteria: 
■ The app displays both "Current Points" (spendable) and "Total Points" (lifetime). 
■ My profile shows my rank (e.g., Bronze, Silver, Gold) which is determined by my 
"Total Points". 
■ Higher ranks unlock access to better discounts in the rewards catalog. 
● US-009: Leaderboards 
○ Story: As a user, I want to see my position on weekly and all-time leaderboards so that I 
can compare my recycling performance with other users. 
○ Acceptance Criteria: 
■ The app has separate tabs for Weekly and All-Time leaderboards. 
■ My own position is highlighted for easy visibility. 
■ The leaderboard displays usernames and their total points for the period. 
● US-010: Badges & Achievements 
○ Story: As a user, I want to earn badges for reaching recycling milestones so that I feel a 
sense of achievement and progress. 
○ Acceptance Criteria: 
■ A dedicated section in my profile displays the badges I've earned and those yet to 
be unlocked. 
■ I receive an in-app notification when I unlock a new badge (e.g., "First 100 
Bottles!"). 
Epic: On-Demand Collection (V1) 
As a busy user, I want the option to have my recyclables collected from my home so that I can 
participate even when I can't get to a kiosk. 
● US-011: Request Delegate Service 
○ Story: As a premium user, I want to request a delegate to pick up recyclables from my 
address so that I can recycle with maximum convenience. 
○ Acceptance Criteria: 
■ I can schedule a pickup time and specify my location. 
■ The app confirms the request and provides an estimated time of arrival for the 
delegate. 
■ The feature is clearly marked as a premium service with associated costs or 
conditions.

1.0 Introduction & Project Vision 
This document marks the official launch and unified action plan for our graduation project, The 
RE Platform. It serves as our single source of truth for our vision, strategy, execution plan, and 
feasibility analysis. 
Our mission is to build a digital ecosystem that fundamentally changes how people interact with 
recycling. We are not just building an app; we are architecting a sustainable, habit-forming 
engine. This initiative is strategically positioned at the intersection of technology, environmental 
sustainability, and economic incentives, directly aligning with the forward-thinking goals of 
Egypt's Vision 2030 and Saudi Arabia's Vision 2030. 
The Problem: Public engagement in recycling is inconsistent. The process is often perceived as 
inconvenient, unrewarding, and lacking a clear feedback loop. 
Our Solution: A gamified, mobile-first platform that rewards users with tangible value 
(discounts) for their recycling efforts, facilitated by a network of accessible smart kiosks and an 
on-demand collection service. 
The Mission: To make recycling an intuitive, rewarding, and integral part of daily life for millions.

To elevate our project from a functional tool to an engaging platform, we propose the following 
enhancements for our V1 roadmap: 
1. Gamification & Ranking Layer: 
○ Dual-Point System & Ranks (e.g., Bronze, Silver, Gold): Implement Current Points 
(for redeeming discounts) and Total Points (a lifetime score). Higher ranks unlock better 
discounts. 
○ Leaderboards (Weekly/All-Time): Foster friendly competition. 
○ Badges & Achievements: Reward milestones (e.g., "First 100 Bottles," "Recycling 
Streak"). 
2. On-Demand Collection Service ("Delegate Request"): 
○ A premium feature allowing users to request a delegate to collect recyclables directly 
from their homes. 
3. Referral & Growth Program: 
○ A system where users invite friends with a unique code/link, and both parties receive 
bonus points. 
4. Community & Impact Hub: 
○ An in-app dashboard showing the community's collective environmental impact. 
5. Corporate Social Responsibility (CSR) Portal (): 
○ A simple web portal for businesses to sign up as partners, manage their discount 
offerings, and track metrics. 
(Note: This is a future goal and is out of scope for the current plan). 
3.0 System Architecture & Technology Stack 
The ReWard platform is built on a modular, service-oriented architecture to ensure scalability, 
maintainability, and high availability. 
3.1 System Architecture 
1. Frontend Layer: 
○ Mobile Application (Native Android): A native Android application will be developed 
using Kotlin/Java to ensure optimal performance, responsiveness, and user experience 
on Android devices. 
○ Web Application (Static Page): Serves as the landing page for marketing, user 
guidance, and partner onboarding. 
2. Backend Layer: 
○ Django REST Framework (DRF): Provides secure API endpoints for all business logic. 
○ Celery + Redis: Handle asynchronous background tasks like referral bonuses, rank 
upgrades, and notification scheduling. 
○ PostgreSQL Database: A centralized relational database ensuring data accuracy, 
scalability, and high performance. 
3. Infrastructure Layer: 
○ Docker & Docker Compose: Used for containerization to ensure portability and 
consistency across development, staging, and production environments. 
○ Nginx (Reverse Proxy): Manages web traffic efficiently and provides HTTPS 
termination for secure communication. 
○ Hostinger Cloud Platform: Provides reliable hosting infrastructure for both web and 
backend services. 
○ GitHub Actions (CI/CD): Automates build, testing, and deployment pipelines for 
continuous integration and delivery.

7.0 Operational & Economic Feasibility 
7.1 Operational Flow 
1. User Interaction: Users download the app, create an account, and receive a unique QR 
code. 
2. Kiosk Interaction: Users scan their QR code at a smart kiosk and deposit recyclables. The 
kiosk weighs the materials and communicates with the backend API. 
3. Backend Processing: The Django backend processes the transaction, calculates points 
(Current and Total), and updates the user's rank and badges. 
4. Reward Redemption: Users browse the rewards catalog and redeem points for discounts 
at partner businesses.
